# 第 9 部分：现实世界中的 Agentic 系统（深入剖析）



---

到目前为止，我们已经涵盖了构成智能体的所有要素：
**工具**、**规划**、**RAG**、**记忆**、**结构**以及多智能体设置中的**协调**。

但你可能会想：
> “这一切在现实世界中究竟体现在哪里？”

让我们来看看一些具有**Agentic 行为**的公开系统——就我们所知。

⚠️ **注意：**
这些系统不是开源的。我们不知道它们的确切内部结构。
以下是基于它们外部行为的合理简化——足以理解 Agentic 堆栈在实践中可能如何表现。

---

## **NotebookLM（Google）：基于你自己数据的 Agentic 搜索**

Google 的 NotebookLM 就像一个个人研究助手。你上传文件，它帮助你处理这些文件——总结、回答问题，甚至生成音频版本或学习指南。

**核心焦点：** 基于你内容的问答——本质上是一个规模化的个人 RAG 系统。

**它可能的工作方式：**
1. **用户上传文件**（PDF、笔记、幻灯片等）
2. **预处理** — 存储它们以便稍后检索。
3. **用户提出问题** — 例如，_“我第二季度战略报告中的关键见解是什么？”_
4. **规划** — 解释任务类型（总结、问答、比较？），识别相关文档/章节。
5. **RAG** — 检索最相关的文档片段。
6. **LLM 生成** — 基于你的内容清晰地响应。
7. **记忆** —
   - 短期：跟踪对话。
   - 长期：可能很少或没有。
8. **工具** — 可能包括文件查看器、总结模块。

**使其具有 Agentic 特性的原因：** 解释目标、搜索你的数据并组合响应——而不仅仅是静态输出。

---

## **Perplexity：基于开放网络的 Agentic 搜索**

Perplexity 为你提供带有来源的直接、类答案响应——而不是一页链接。

**它可能的工作方式：**
1. **用户提出问题** — 例如，_“关于阿尔茨海默病治疗的最新研究是什么？”_
2. **规划** — 解释意图（“最新”、“可信”），决定搜索方法。
3. **工具使用** — 通过 Web API 发出查询。
4. **RAG** — 检索相关页面片段。
5. **LLM 响应** — 合成带有引用的答案。
6. **记忆** —
   - 短期：会话上下文。
   - 长期：可能存储偏好（例如，“始终使用 WSJ 作为新闻来源”）。

**使其具有 Agentic 特性的原因：** 获取信息、决定使用什么，并在多步骤循环中构建答案。

---

## **DeepResearch（OpenAI）：深度 Agentic 工作流**

DeepResearch 处理**开放式、复杂的研究任务**——例如，市场分析、竞争格局、技术深入研究。

**它可能的工作方式：**
1. **用户提出广泛任务** — 例如，_“分析教育初创公司的生成式 AI 格局。”_
2. **规划** — 将其分解为子任务（资金、趋势、公司、风险），形成执行计划。
3. **工具** — 可能包括：
   - Web 搜索
   - 文档阅读器（PDF）
   - 数据工具（电子表格、图表）
   - 报告生成模块
4. **Agentic RAG** — 不是一次性检索——随着任务的发展进行获取、反思、重新获取。
5. **记忆** —
   - 情景记忆：跟踪哪些部分已完成。
   - 语义记忆：存储关键事实/名称。
6. **多步骤推理** — 循环：规划 → 检索 → 读取 → 重新思考 → 生成 → 细化 → 重复。

**使其具有 Agentic 特性的原因：** 大量规划、迭代工具使用、自我导向的进展。
